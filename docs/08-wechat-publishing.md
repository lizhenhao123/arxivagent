# 08 Wechat Publishing

> 当前说明：该模块降级为 V2 预留，不纳入第一批代码实现。

## 目标

定义微信公众号草稿、素材上传、发布、状态查询和失败重试逻辑，作为 V2 发布模块实现依据。

## 主要能力

- 获取并缓存 `access_token`
- 上传素材
- 创建草稿
- 发起发布
- 轮询发布状态
- 记录失败原因

## V1 当前决策

- 公众号只接一个账号
- 目前不接入真实公众号
- V1 先把结果保存到站内页面，人工审阅后手动发布到公众号
- 正文主格式为 Markdown，不直接生成微信最终稿

## 发布链路

```text
生成 HTML 草稿 -> 上传素材 -> 创建微信草稿 -> 提交发布 -> 查询发布状态 -> 回写结果
```

## 关键记录字段

- `draft_media_id`
- `publish_id`
- `publish_status`
- `wechat_errcode`
- `wechat_errmsg`
- `failed_reason`

## 约束

- 不以接口提交成功作为最终发布成功
- 发布必须支持重试
- 敏感密钥不能明文暴露

## 推荐失败处理

- 临时失败：自动重试
- 配置错误：中止并提醒人工
- 内容错误：退回审核

## 与代码生成的关系

本文件会直接影响：

- WeChat client
- token cache
- publish job worker
- 发布状态轮询逻辑

## 需要你补充的信息

- 是订阅号还是服务号
- 是否已有公众号测试环境
- 是否需要预约发布时间
- 封面图是否由系统生成还是人工上传
